# Plain Text Sanitizer

A small static web app for cleaning pasted text before copying it into Gmail or another destination.

The app has no build step. It can be hosted directly on GitHub Pages.

## GitHub Pages Deployment

1. Create a new GitHub repository.
2. Copy these files into the repository root:
   - `index.html`
   - `styles.css`
   - `app.js`
   - `README.md`
   - `.nojekyll`
3. Commit and push the files.
4. In the repository settings, enable GitHub Pages for the main branch and root folder.
5. Open the Pages URL once GitHub finishes publishing.

## Design Doc

### 1. Purpose

Build a small web app that takes drafted text as input and returns sanitized output. The app removes visible and hidden formatting artifacts, normalizes punctuation, standardizes line breaks internally, and can copy the result as Gmail-oriented HTML.

### 2. Core Goal

The core goal is to convert formatted text into predictable, sanitized content while preserving the user’s intended words and paragraph breaks.

### 3. Non-Goals

The app does not rewrite meaning.

The app does not detect whether text was written by a person or generated by software.

The app does not modify tone, style, vocabulary, or sentence structure.

The first version focuses on sanitization and Gmail-compatible copy behavior.

### 4. Target User Flow

1. User opens the web app.
2. User pastes text into the input box.
3. The app immediately shows sanitized output.
4. User reviews the output.
5. User clicks **Copy for Gmail** to copy sanitized Verdana HTML, or **Copy plain text** for plain text.
6. User pastes the result into Gmail.

### 5. Main Screen

The app uses one page with:

- Original text box
- Clean text preview
- Preset selector
- Cleanup rule checkboxes
- Copy plain text button
- Copy for Gmail button
- Clear button
- Inspector panel

The desktop layout places the original text, clean text, and controls side by side. The mobile layout stacks them.

### 6. Sanitization Pipeline

#### Step 1: Read input as plain text

The input area is a normal `textarea`. On paste, the app requests `text/plain` from the clipboard and inserts that text into the textarea.

#### Step 2: Normalize line endings

The sanitizer converts:

- `U+000D U+000A` to `U+000A`
- `U+000D` to `U+000A`

#### Step 3: Remove hidden characters

The sanitizer removes known invisible or formatting characters, including:

- `U+00AD` SOFT HYPHEN
- `U+034F` COMBINING GRAPHEME JOINER
- `U+061C` ARABIC LETTER MARK
- `U+180B` through `U+180E`
- `U+200B` through `U+200F`
- `U+202A` through `U+202E`
- `U+2060` through `U+206F`
- `U+FE00` through `U+FE0F`
- `U+FEFF` BYTE ORDER MARK
- `U+FFF9` through `U+FFFB`
- `U+E0000` through `U+E007F`

#### Step 4: Normalize spaces

The sanitizer converts unusual spaces to `U+0020 SPACE`, including:

- `U+00A0` NO-BREAK SPACE
- `U+1680` OGHAM SPACE MARK
- `U+2000` through `U+200A`
- `U+202F` NARROW NO-BREAK SPACE
- `U+205F` MEDIUM MATHEMATICAL SPACE
- `U+3000` IDEOGRAPHIC SPACE

Trailing spaces are removed from each line.

Repeated internal spaces are preserved by default. The user can enable a stricter option to collapse repeated spaces.

#### Step 5: Normalize punctuation

The sanitizer converts typographic punctuation into common keyboard punctuation.

Default replacements:

- Quote-like single marks become `'` (`U+0027 APOSTROPHE`).
- Quote-like double marks become `"` (`U+0022 QUOTATION MARK`).
- En dash-like characters become `-`.
- Em dash-like characters become ` -- `.
- Ellipsis becomes `...`.
- Line-start bullets become `- `.

#### Step 6: Normalize paragraph breaks

The sanitizer trims leading and trailing blank lines.

Runs of three or more newline characters are reduced to two newline characters.

#### Step 7: Optional Strict ASCII Mode

Strict ASCII mode is optional.

Strict ASCII mode:

- Applies Unicode compatibility decomposition.
- Removes combining marks.
- Removes remaining non-ASCII characters.

This option is off by default because it can alter names, foreign-language words, symbols, and emoji.

### 7. Sanitization Options

Enabled by default in Gmail Safe mode:

- Remove hidden characters
- Normalize line endings
- Normalize unusual spaces
- Trim trailing spaces
- Limit extra blank lines
- Normalize quotes and apostrophes
- Normalize dashes
- Normalize ellipses
- Convert line-start bullets to hyphens

Disabled by default in Gmail Safe mode:

- Collapse repeated spaces
- Convert tabs to spaces
- Remove emoji
- Strict ASCII only

### 8. Output Guarantees

The **Copy plain text** button writes the sanitized string using plain-text clipboard output.

The **Copy for Gmail** button builds Gmail-compatible HTML from the sanitized string and writes that exact HTML string to the clipboard as `text/html`. It does not intentionally add hidden characters to the Gmail HTML payload.

The Gmail HTML output is modeled on Gmail's own default Verdana compose markup:

- one outer wrapper `div`
- one `div.gmail_default` per text line
- `style="font-family: verdana, sans-serif;"` on every Gmail paragraph div
- no intentionally added zero-width spaces or other hidden characters
- escaped text content
- `<div class="gmail_default" style="font-family: verdana, sans-serif;"><br></div>` for blank lines
- `<br>` at the end of the final non-empty paragraph div
- final `<br clear="all">` inside the outer wrapper

The target shape is:

```html
<div><div class="gmail_default" style="font-family: verdana, sans-serif;">Paragraph one</div><div class="gmail_default" style="font-family: verdana, sans-serif;"><br></div><div class="gmail_default" style="font-family: verdana, sans-serif;">Paragraph two<br></div><br clear="all"></div>
```

The HTML output is generated from the sanitized string, not from the original pasted source.

### 9. Inspector

### 10. Diff View

Version 1 does not include a full side-by-side visual diff.

The inspector provides counts and remaining character warnings.

A later version can add a diff panel that labels removed or replaced code points.

### 11. Presets

#### Gmail Safe

Default preset.

Behavior:

- Remove hidden characters
- Normalize punctuation
- Normalize whitespace
- Preserve paragraphs
- Preserve non-ASCII letters, symbols, and emoji unless they are hidden formatting characters

#### Strict Plain Text

Behavior:

- Everything in Gmail Safe
- Convert tabs to spaces
- Collapse repeated spaces
- Remove emoji and pictographic symbols
- Enforce Strict ASCII mode

#### Light Cleanup

Behavior:

- Remove hidden characters
- Normalize line endings
- Normalize unusual spaces
- Trim trailing spaces
- Leave punctuation mostly unchanged

### 12. Clipboard Behavior

The app has two copy buttons.

**Copy plain text** writes the sanitized output as `text/plain`.

**Copy for Gmail** writes a Gmail-compatible HTML string directly to the clipboard. The Gmail-oriented clipboard payload is `text/html`, not a textarea copy.

The clean output preview is a rendered `div`, not a textarea. It is displayed as `verdana, sans-serif` at Gmail normal size.

### 13. Privacy Requirements

The app runs fully in the browser.

The app has no server-side component.

The app does not include analytics.

The app does not store drafts.

### 14. Technical Architecture

The project is a static frontend with:

- `index.html` for structure
- `styles.css` for layout and appearance
- `app.js` for the sanitizer, Gmail HTML generation, and UI bindings

The sanitizer is a pure function:

```js
sanitize(inputText, options)
```

It returns:

```js
{
  cleanText,
  changes,
  stats,
  warnings,
  remainingNonAscii,
  options
}
```

### 15. Data Model

Input:

- `inputText`
- `options`

Output:

- `cleanText`
- `changes`
- `stats`
- `warnings`
- `remainingNonAscii`
- `options`

Each change record has:

- `type`
- `count`

### 16. Example Transformations

Examples show both visible text and exact code-point representation. Some problematic characters are invisible or visually ambiguous, so examples must not rely on visual comparison alone.

#### Example 1: Curly quotes and curly apostrophe

Input display:

```text
“Don’t send this yet,” she said.
```

Input character representation:

```text
U+201C Don U+2019 t send this yet, U+201D she said.
```

Sanitized output display:

```text
"Don't send this yet," she said.
```

Sanitized output character representation:

```text
U+0022 Don U+0027 t send this yet, U+0022 she said.
```

Changes:

- `U+201C LEFT DOUBLE QUOTATION MARK` -> `U+0022 QUOTATION MARK`
- `U+201D RIGHT DOUBLE QUOTATION MARK` -> `U+0022 QUOTATION MARK`
- `U+2019 RIGHT SINGLE QUOTATION MARK` -> `U+0027 APOSTROPHE`

#### Example 2: Em dash and en dash

Input display:

```text
The meeting is tomorrow—unless the Monday–Friday schedule changes.
```

Input character representation:

```text
The meeting is tomorrow U+2014 unless the Monday U+2013 Friday schedule changes.
```

Sanitized output display:

```text
The meeting is tomorrow -- unless the Monday-Friday schedule changes.
```

Sanitized output character representation:

```text
The meeting is tomorrow U+0020 U+002D U+002D U+0020 unless the Monday U+002D Friday schedule changes.
```

Changes:

- `U+2014 EM DASH` -> `U+0020 SPACE + U+002D HYPHEN-MINUS + U+002D HYPHEN-MINUS + U+0020 SPACE`
- `U+2013 EN DASH` -> `U+002D HYPHEN-MINUS`

#### Example 3: Ellipsis

Input display:

```text
I was thinking… maybe we should wait.
```

Input character representation:

```text
I was thinking U+2026 maybe we should wait.
```

Sanitized output display:

```text
I was thinking... maybe we should wait.
```

Sanitized output character representation:

```text
I was thinking U+002E U+002E U+002E maybe we should wait.
```

Change:

- `U+2026 HORIZONTAL ELLIPSIS` -> `U+002E U+002E U+002E`

#### Example 4: Non-breaking space


### 17. Edge Cases

The default preset avoids aggressive changes that could damage meaning.

Examples that require care:

- Names that contain accents
- Foreign-language text
- Legal text with section symbols
- Math symbols
- Emoji used intentionally
- Code blocks
- Markdown lists
- Quoted email replies
- URLs
- Apostrophes inside contractions
- Quotation marks inside quoted speech
