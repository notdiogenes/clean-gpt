# Plain Text Sanitizer

A small static web app for cleaning pasted text before copying it into Gmail, Google Docs, Microsoft Word, forms, or other editors.

The app has no build step. It can be hosted directly on GitHub Pages.

## GitHub Pages Deployment

1. Create a new GitHub repository.
2. Copy these files into the repository root:
   - `index.html`
   - `styles.css`
   - `app.js`
   - `README.md`
   - `.nojekyll`
3. Commit and push the files.
4. In the repository settings, enable GitHub Pages for the main branch and root folder.
5. Open the Pages URL once GitHub finishes publishing.

## Design Doc

### 1. Purpose

Build a browser-only copy/paste cleanup tool that separates source cleanup from destination formatting.

The app first removes hidden and source-specific artifacts from pasted text. Then it shapes the output for the selected paste destination.

### 2. Core Goal

The core goal is predictable output:

- clean the source text
- preserve the user’s intended words and paragraph breaks
- match the selected destination’s expected plain-text or HTML behavior
- show exactly what character replacements were made

### 3. Non-Goals

The app does not rewrite meaning.

The app does not detect whether text was written by a person or generated by software.

The app does not modify tone, style, vocabulary, or sentence structure.

The app does not try to be a rich text editor.

### 4. Critical UX Decision

The app no longer uses a rendered output div as the main review surface.

The output is a single read-only textarea because that makes the exact copied characters visible. Gmail does not need a visible Gmail-preview div because the Gmail button generates the Gmail-compatible HTML clipboard payload directly.

### 5. Target User Flow

1. User opens the web app.
2. User pastes text into the input box.
3. The app sanitizes the source text.
4. User chooses a destination:
   - Gmail
   - Google Docs
   - Microsoft Word
   - Plain text / forms
5. The output textarea updates to show the destination-ready text.
6. User reviews the Inspector for exact character replacements.
7. User clicks the destination copy button.
8. User pastes into the target app.

### 6. Main Screen

The app uses one page with:

- Original text box
- Destination selector
- Destination-ready output textarea
- Copy visible text button
- Copy for selected destination button
- Base cleanup preset selector
- Cleanup rule checkboxes
- Inspector with counts, warnings, remaining non-ASCII characters, and exact replacement details

### 7. Destination Modes

#### Gmail

Gmail mode keeps keyboard punctuation in the visible output textarea.

The destination copy button writes a Gmail-shaped `text/html` clipboard payload directly. It does not copy from the visible textarea or from a rendered preview div.

The Gmail HTML shape is:

```html
<div><div class="gmail_default" style="font-family: verdana, sans-serif;">Paragraph one</div><div class="gmail_default" style="font-family: verdana, sans-serif;"><br></div><div class="gmail_default" style="font-family: verdana, sans-serif;">Paragraph two<br></div><br clear="all"></div>
```

The app does not intentionally add zero-width spaces to the Gmail HTML payload.

#### Google Docs

Google Docs mode converts the cleaned keyboard punctuation into document-style smart punctuation in the visible output textarea.

The destination copy button writes `text/plain` only, so Google Docs can apply the current document style when pasted.

Destination typography transformations include:

- `"` to opening or closing curly double quotes
- `'` to curly apostrophe or curly single quotes
- ` -- ` to em dash
- `...` to ellipsis

#### Microsoft Word

Microsoft Word mode uses the same document-style smart punctuation as Google Docs mode.

The destination copy button writes `text/plain` only, so Word can apply the current document style when pasted.

#### Plain text / forms

Plain text mode keeps keyboard-safe punctuation and writes only `text/plain`.

### 8. Sanitization Pipeline

#### Step 1: Read input as plain text

The input area is a normal `textarea`. On paste, the app requests `text/plain` from the clipboard and inserts that text into the textarea.

#### Step 2: Normalize line endings

The sanitizer converts:

- `U+000D U+000A` to `U+000A`
- `U+000D` to `U+000A`

#### Step 3: Remove hidden characters

The sanitizer removes known invisible or formatting characters, including:

- `U+00AD` SOFT HYPHEN
- `U+034F` COMBINING GRAPHEME JOINER
- `U+061C` ARABIC LETTER MARK
- `U+180B` through `U+180E`
- `U+200B` through `U+200F`
- `U+202A` through `U+202E`
- `U+2060` through `U+206F`
- `U+FE00` through `U+FE0F`
- `U+FEFF` BYTE ORDER MARK
- `U+FFF9` through `U+FFFB`
- `U+E0000` through `U+E007F`

#### Step 4: Normalize spaces

The sanitizer converts unusual spaces to `U+0020 SPACE`, including:

- `U+00A0` NO-BREAK SPACE
- `U+1680` OGHAM SPACE MARK
- `U+2000` through `U+200A`
- `U+202F` NARROW NO-BREAK SPACE
- `U+205F` MEDIUM MATHEMATICAL SPACE
- `U+3000` IDEOGRAPHIC SPACE

Trailing spaces are removed from each line.

Repeated internal spaces are preserved by default. The user can enable a stricter option to collapse repeated spaces.

#### Step 5: Normalize source punctuation

The sanitizer converts source typographic punctuation into keyboard punctuation before destination formatting is applied.

Default source replacements:

- Quote-like single marks become `'` (`U+0027 APOSTROPHE`).
- Quote-like double marks become `"` (`U+0022 QUOTATION MARK`).
- En dash-like characters become `-`.
- Em dash-like characters become ` -- `.
- Ellipsis becomes `...`.
- Line-start bullets become `- `.

#### Step 6: Apply destination typography

After source cleanup, the app applies destination typography.

Gmail and Plain text modes keep keyboard punctuation.

Google Docs and Microsoft Word modes convert keyboard punctuation to document-style smart punctuation.

#### Step 7: Normalize paragraph breaks

The sanitizer trims leading and trailing blank lines.

Runs of three or more newline characters are reduced to two newline characters.

#### Step 8: Optional Strict ASCII Mode

Strict ASCII mode is optional.

Strict ASCII mode:

- Applies Unicode compatibility decomposition.
- Removes combining marks.
- Removes remaining non-ASCII characters.

This option is off by default because it can alter names, foreign-language words, symbols, and emoji.

### 9. Sanitization Options

Enabled by default in Gmail Safe mode:

- Remove hidden characters
- Normalize line endings
- Normalize unusual spaces
- Trim trailing spaces
- Limit extra blank lines
- Normalize source quotes and apostrophes
- Normalize source dashes
- Normalize source ellipses
- Convert line-start bullets to hyphens

Disabled by default in Gmail Safe mode:

- Collapse repeated spaces
- Convert tabs to spaces
- Remove emoji
- Strict ASCII only

### 10. Output Guarantees

The output textarea shows the exact visible text used for the selected destination.

The **Copy visible text** button always writes the output textarea value as `text/plain`.

The selected destination copy button behaves as follows:

- Gmail: writes Gmail-compatible `text/html`
- Google Docs: writes destination-ready `text/plain`
- Microsoft Word: writes destination-ready `text/plain`
- Plain text / forms: writes destination-ready `text/plain`

The Gmail HTML output is generated from the destination-ready string, not from the original pasted source and not from the rendered DOM.

### 11. Inspector

The inspector shows:

- Input character count
- Output character count
- Hidden characters removed
- Spaces normalized
- Quotes normalized
- Dashes normalized
- Ellipses normalized
- Bullets converted
- Trailing spaces removed
- Blank-line runs reduced
- Destination typography changes
- Remaining non-ASCII characters
- Remaining suspicious characters

The Inspector also lists exact replacement details by Unicode code point and name.

### 12. Example Transformations

#### Gmail destination

Input display:

```text
“Don’t wait—send it…”
```

Sanitized source text:

```text
"Don't wait -- send it..."
```

Gmail destination output:

```text
"Don't wait -- send it..."
```

Destination behavior:

- copied as Gmail-compatible Verdana HTML
- keyboard quotes and apostrophes remain keyboard quotes and apostrophes
- em dash is represented as ` -- `
- ellipsis is represented as `...`

#### Google Docs / Word destination

Input display:

```text
“Don’t wait—send it…”
```

Sanitized source text:

```text
"Don't wait -- send it..."
```

Document destination output:

```text
“Don’t wait—send it…”
```

Destination behavior:

- copied as text/plain
- smart quotes are restored as destination typography
- ` -- ` becomes an em dash
- `...` becomes an ellipsis

#### Hidden character cleanup

Input character representation:

```text
Please re U+200B view this before sending.
```

Output display:

```text
Please review this before sending.
```

Change detail:

```text
U+200B ZERO WIDTH SPACE -> removed
```
